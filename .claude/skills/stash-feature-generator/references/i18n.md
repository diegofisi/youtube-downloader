# i18n (es/en) — minimalista, sin diccionarios

Cada texto lleva su par inline. No hay archivos de traducción ni claves.

## Texto generado desde TS: `t(es, en)`

```ts
import { t } from '../../core/i18n';
showToast(t('Añadido a la cola', 'Added to queue'), it.title, 'done');
btn.innerHTML = `${I.spinner} ${t('Analizando…', 'Analyzing…')}`;
```
`t()` evalúa `getLang()` en el MOMENTO de la llamada. Español primero, inglés segundo, siempre.

## HTML estático (index.html): atributos `data-en*`

El HTML se escribe en español; `applyStaticI18n()` (llamada primera en main.ts) lo traduce si
`getLang() === 'en'`:

| Atributo | Reemplaza | Ejemplo real |
|---|---|---|
| `data-en` | El texto del elemento (solo nodos de texto si tiene hijos: no destruye SVGs) | `<button id="btn-analyze" data-en="Preview videos">Previsualizar</button>` |
| `data-en-ph` | El `placeholder` | `<input id="search-input" placeholder="Buscar en YouTube…" data-en-ph="Search YouTube…">` |
| `data-en-title` | El `title` (tooltip) | `<button title="Minimizar" data-en-title="Minimize">` |

Todo elemento nuevo de index.html con texto visible DEBE llevar su `data-en*`.

## Records/constantes con texto: getters (evaluación perezosa)

Un `Record` a nivel de módulo se evalúa al importar, ANTES de saber el idioma… salvo que uses
getters. Patrón obligatorio (STATUS_META, QMAP, Q_LABEL):

```ts
export const STATUS_META: Record<Status, { readonly label: string; … }> = {
  ok: {
    get label() { return t('Descargable', 'Downloadable'); },   // ✅ se evalúa al leer
    color: 'var(--success)',
  },
};
// ❌ NUNCA: ok: { label: t('Descargable', 'Downloadable') }  ← queda congelado en un idioma
```
Alternativa para arrays: función perezosa — `const MODE_DEFS = () => [ … ]` (descargar.ts),
`const getSteps = () => [ … ]` (onboarding.ts, "para no depender del orden de import de i18n").

## `setLang` recarga la app

`setLang(l)` persiste en localStorage (`stash.lang`) y hace `location.reload()`. Por eso NO hay
que re-renderizar nada al cambiar idioma: el selector de Ajustes simplemente llama a setLang.
No construyas mecanismos de "re-traducción en caliente".

## Qué NO traducir

- Formatos y valores técnicos: `MP4`, `MKV`, `WebM`, `MP3`, `M4A`, `Opus`, `4K`, `1080p`, bitrates.
- Nombres de producto/marca: Stash, YouTube, yt-dlp, ffmpeg, deno.
- Plantillas de yt-dlp (`%(title)s [%(id)s]`), URLs, rutas, ids, claves de localStorage.
- Mensajes de error del BACKEND: van solo en español (Result<_, String>); el frontend los muestra tal cual.
- Textos idénticos en ambos idiomas se escriben igual dos veces: `t('Video + audio', 'Video + audio')`.

## Fechas y locales: `getLang()`

Para formateo dependiente de idioma se pasa el lang como locale (shared/lib/format.ts):

```ts
new Date(secs * 1000).toLocaleString(getLang(), { day: '2-digit', month: 'short', … });
```
`timeAgo` usa `t()` para sus unidades ("hace 5 min" / "5 min ago"). En tests, sin localStorage
el default es `'es'`: las cadenas esperadas son las españolas (format.test.ts).
